# 2026-06-04: FullSnap Becomes The Main Public-Good Repo

I cleaned up the public GitHub direction around one useful public repo instead of keeping older experiments in the foreground.

## What Changed

- Strengthened [`fullsnap-extension`](https://github.com/Najariya/fullsnap-extension) as the main public-good project.
- Added clearer public docs for architecture, privacy, status, roadmap, security, and license.
- Kept the privacy policy live through GitHub Pages.
- Archived older public prototypes that no longer represent the active direction.

## Why It Matters

FullSnap is a small example of the kind of work I want to make more visible: practical, privacy-first, understandable, and useful enough for someone else to inspect or build on.

The rule is simple: not every experiment needs to be public-good work. A repo earns that label when it is clear, safe, documented, and useful.
